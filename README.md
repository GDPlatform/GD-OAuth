# GDPlatform - OAuth (Geode Mod)

**GD-OAuth** is the official bridge between Geometry Dash and the **GDPlatform** ecosystem. It allows developers to securely verify player identities using the Geode infrastructure.

---

## Getting Started

### 1. Create Your Account

To start using the system, you first need an account on our platform:

* **Register:** [https://gdplatform.netlify.app/register](https://gdplatform.netlify.app/register)
* **Login:** [https://gdplatform.netlify.app/login](https://gdplatform.netlify.app/login)

### 2. Link Your GD Account

Once registered, linking your account is simple:

1. Open Geometry Dash with the mod installed.
2. Generate a verification code from the Mod menu.
3. Enter the code on the [Verification](https://gdplatform.netlify.app/verification-step) page.
4. **Done!** Your GD `AccountID` is now linked to your GDPlatform account.

---

## API Documentation

If you’re a developer, you can integrate GDPlatform identity into your own services.

**API Domain:** `api--gdplatform.netlify.app`

### Status Polling

Check whether a user has completed the linking process using the code generated by the Mod.

* **Endpoint:** `GET /geodesdk/gd-oauth/status/:code`
* **Response:**

```json
{ "status": "pending" | "completed", "linkedTo": "uid_firebase" }
```

### OAuth Exchange (External Apps)

Use this endpoint to exchange the code obtained from `/connect` for the user’s real data.

* **Endpoint:** `POST /oauth/code`
* **Headers:** `Content-Type: application/json`
* **Body:**

```json
{
  "code": "auth_xyz123",
  "scopes": ["account_id", "profile"]
}
```

---

## Security & CORS

All our APIs support **Cross-Origin** requests and are protected by Netlify security headers. If you encounter a `preflight` error, make sure to explicitly include the `Content-Type: application/json` header.
